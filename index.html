GameRun
Rent it all out!

A CRUD app that allows you to rent board games for your events!


Technical Skills Used:

  • Node.js
  • React.js
  • Express
  • MVC

What is it?

This project is based on the User being able to rent board games and the Admin being able to add the games to the database (or inventory if that's what you want to call it ^^).


Admin

Lets say that the Admin is the owner of a board game shop.... Admin purchased a new board game and want to add it in their inventory so that Users can rent it out.

Instead of putting in the details of the board game manually, Admin can just search for the game that they purchased and the application will generate an auto-filled form to add the game into their inventory!



Admin is able to edit the details and can always delete the board games, and keep track of the orders!



User

As for the Users, the homepage will state if the board game is available for rent or not.



Users can then choose a package to start renting the games and view their own orders after everything is complete!



Learning Process

Unlike my Munch! Project, I started building this application by drawing out some wireframes and users' stories. It took me some time to get it done but it was all worth it! It made my developing process much easier as I know what I had to do next.

Initially, the Admin had to fill up the form manually when they want to add a game but I realized that it was not ideal. So, I decided to utilize BoardGameGeek's API and generate the form with auto-filled details, and it seems like the flow was better! It took me awhile to figure out how to implement it as the information extracted was in an XML format.

It was our first time implementing the MVC pattern and it was actually challenging to do this project while trying to understand how the concept works. But oh wells, Challenges are what make life interesting and overcoming then is what makes life meaningful they say.


Tripper
Manage your trips with this application!

Technical Skills Used:

  • Ruby on Rails

What is it?

My teammates (Neel & Vivek) and I love to travel so we thought of building an application to keep track of our trips. We came up with the features we want to have in the app such as our flight details, to-do lists and expenses.

We were thinking on how to make the application a little different and we came out with the idea of auto-generating emergency numbers of the country you're visiting. Also, this application is recommended to be used on mobile due to the design.



Learning Process

I was in-charge of the flight details and SOS number feature, and the design of the application. It was so cool to actually see a whole application being built by different features that was done by different people.

Even though it was challenging to bring everyone's codes together, it was interesting to learn and read different kind of methods on how others write their codes.
